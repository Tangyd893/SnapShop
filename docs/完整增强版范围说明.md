# SnapShop 完整增强版范围说明

## 一、文档说明

本文档定义 SnapShop **完整增强版**的交付边界、必做能力、对应设计文档与任务编号。动工前须以本文档为准，避免仍按「最小可用版」裁剪功能。

版本关系：

| 版本 | 说明 |
| --- | --- |
| 最小可用版 | 跑通登录 → 商品 → 秒杀 → MQ 下单 → 查结果 → 订单详情 |
| 稳定演示版 | 在最小版基础上补齐可靠性（Lua、幂等、本地消息表、死信、补偿、压测） |
| **完整增强版** | 在稳定演示版基础上补齐管理后台、模拟支付全流程、可观测性、自动化部署 |

**支付说明**：完整增强版采用**模拟支付**（非真实第三方支付）；真实支付接入列为后续迭代，不在本版范围。

## 四、非功能指标（完整增强版验收）

### 4.1 秒杀压测 SLA（T-905）

| 指标 | 目标值 | 环境 |
| --- | --- | --- |
| 秒杀提交接口 P99 延迟 | ≤ 200ms | 本地 Docker，4C8G，100 并发 |
| 秒杀成功率（有库存时） | ≥ 99% | 活动库存 ≥ 并发数 |
| 超卖 | 0 | 活动库存 100，并发 500 |
| 重复下单 | 0 | 同一用户同一活动商品 |

### 4.2 消息可靠性

| 配置项 | 默认值 | 配置键（Nacos） |
| --- | --- | --- |
| 消费最大重试次数 | 5 | `snapshop.rabbitmq.consumer.max-retries` |
| 死信进入阈值 | 同上 | 与重试次数一致 |
| 本地消息补偿扫描间隔 | 30 秒 | `snapshop.message.compensate.interval-seconds` |
| 本地消息最大重试 | 10 | `snapshop.message.compensate.max-retries` |

详见 [RabbitMQ可靠消息设计文档.md](RabbitMQ可靠消息设计文档.md) §7.3、§九。
