# SnapShop 工程配置说明

## 一、文档说明

本文档锁定 SnapShop 项目的技术版本、服务端口、Nacos 配置命名、Docker Compose 与数据库脚本约定。实现时以本文档为准，避免「17 或 21」「JMeter 或 Gatling」等歧义。

---

## 二、技术版本（已锁定）

| 组件 | 版本 | 说明 |
| --- | --- | --- |
| Java | **21**（LTS） | `maven.compiler.release=21` |
| Maven | 3.9.6+ | 父 POM 统一管理 |
| Spring Boot | **3.2.5** | 父 POM `spring-boot.version` |
| Spring Cloud | **2023.0.1** | 与 Boot 3.2 对齐 |
| Spring Cloud Alibaba | **2023.0.1.0** | Nacos、Sentinel 等 |
| Node.js | 20 LTS+ | 前端构建 |
| Vue | 3.4+ | 前端框架 |
| Vite | 5.x | 前端构建工具 |
| MySQL | 8.0.36（Docker 镜像 tag） | 业务库 |
| Redis | 7.2（Docker 镜像 tag） | 缓存与秒杀库存 |
| RabbitMQ | 3.13-management | 含管理控制台 |
| Nacos | 2.3.2 | 注册与配置 |
| 压测 | **JMeter** 5.6+ | 脚本目录 `testing/load/` |

父 POM 坐标建议：

```xml
<groupId>com.snapshop</groupId>
<artifactId>snapshop-parent</artifactId>
<version>1.0.0-SNAPSHOT</version>
```

## 四、Nacos 配置约定

### 4.1 命名空间与分组

| 项 | 值 |
| --- | --- |
| 命名空间 | `snapshop-dev`（开发）、`snapshop-prod`（生产） |
| Group | `DEFAULT_GROUP` |
| 配置格式 | YAML |

### 4.2 dataId 列表

每个服务两个 dataId：`{服务名}.yaml`（业务配置）、`{服务名}-datasource.yaml`（数据源，可选合并）。

| dataId | 内容 |
| --- | --- |
| `snapshop-gateway.yaml` | 路由、跨域、限流 |
| `snapshop-auth.yaml` | JWT 密钥、过期时间 |
| `snapshop-user.yaml` | 用户相关业务参数 |
| `snapshop-product.yaml` | 缓存 TTL |
| `snapshop-seckill.yaml` | 秒杀令牌 TTL、Redis 键前缀 |
| `snapshop-order.yaml` | 订单超时分钟数 |
| `snapshop-inventory.yaml` | 库存扣减参数 |
| `snapshop-payment.yaml` | 模拟支付开关 |
| `snapshop-common.yaml` | RabbitMQ、Redis、MySQL 连接（各服务 shared 引用） |
| `sentinel-rules-snapshop-gateway-flow.yaml` | Sentinel 网关限流规则（Group: `SENTINEL_GROUP`） |
| `sentinel-rules-snapshop-seckill-flow.yaml` | Sentinel 秒杀流控规则 |
| `sentinel-rules-snapshop-seckill-degrade.yaml` | Sentinel 秒杀熔断规则 |
| `sentinel-rules-snapshop-order-degrade.yaml` | Sentinel 订单熔断规则 |
| `sentinel-rules-snapshop-inventory-degrade.yaml` | Sentinel 库存熔断规则 |

### 4.3 公共配置键（snapshop-common.yaml）

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/snapshop?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
    username: snapshop
    password: ${MYSQL_PASSWORD:snapshop123}
  data:
    redis:
      host: localhost
      port: 6379
  rabbitmq:
    host: localhost
    port: 5672
    username: snapshop
    password: ${RABBITMQ_PASSWORD:snapshop123}

snapshop:
  rabbitmq:
    consumer:
      max-retries: 5
  message:
    compensate:
      interval-seconds: 30
      max-retries: 10
  seckill:
    token-ttl-seconds: 60
    result-ttl-seconds: 86400
  jwt:
    access-token-ttl-seconds: 7200
    refresh-token-ttl-seconds: 604800
```

### 4.4 导入方式

1. 启动 Nacos 后访问 `http://localhost:8848/nacos`。
2. 创建命名空间 `snapshop-dev`。
3. 将 `docker/nacos/init-config/` 下 YAML 文件逐一导入（实现阶段创建该目录）。
4. 各服务 `bootstrap.yml` 中指定 `namespace` 与 `shared-configs`。

## 五、Docker Compose 规格

文件路径：`docker/docker-compose.yml`（实现阶段由 T-007 创建）。

### 5.1 服务列表

| 服务名 | 镜像 | 宿主机端口 | 数据卷 |
| --- | --- | --- | --- |
| mysql | mysql:8.0.36 | 3306 | `./mysql/data` |
| redis | redis:7.2 | 6379 | `./redis/data` |
| rabbitmq | rabbitmq:3.13-management | 5672, 15672 | `./rabbitmq/data` |
| nacos | nacos/nacos-server:v2.3.2 | 8848, 9848 | `./nacos/data` |
| sentinel | bladex/sentinel-dashboard:1.8.8 | **8888**（映射容器 8080） | 无持久化 |

### 5.2 环境变量（.env.example）

```env
MYSQL_ROOT_PASSWORD=root123
MYSQL_DATABASE=snapshop
MYSQL_USER=snapshop
MYSQL_PASSWORD=snapshop123
RABBITMQ_DEFAULT_USER=snapshop
RABBITMQ_DEFAULT_PASS=snapshop123
NACOS_AUTH_ENABLE=false
```

### 5.3 健康检查

- MySQL：`mysqladmin ping`
- Redis：`redis-cli ping`
- RabbitMQ：管理 API `/api/health/checks/alarms`
- Nacos：`/nacos/v1/console/health/readiness`

### 5.4 启动顺序

`mysql` → `redis` → `rabbitmq` → `nacos` → `sentinel`（depends_on + healthcheck）。

## 六、数据库脚本约定

| 文件 | 路径 | 说明 |
| --- | --- | --- |
| 建表脚本 | `docker/mysql/init/01-schema.sql` | 与数据库设计文档表结构一致 |
| 初始数据 | `docker/mysql/init/02-data.sql` | 测试账号、商品、秒杀活动 |
| 迁移脚本 | `docker/mysql/migration/V{版本}__{描述}.sql` | 后续变更 |

Compose 挂载：`./mysql/init:/docker-entrypoint-initdb.d`。

### 6.1 状态字段约定

所有业务状态字段统一：

- 类型：`varchar(32)`
- 存储：英文大写蛇形或驼峰码表，例如 `PENDING_PAY`、`PAID`、`QUEUING`
- **禁止**在同一项目混用字符串与小整数状态

```text
seckill:stock:{activityId}:{skuId}
seckill:user:{activityId}:{skuId}:{userId}
seckill:result:{requestId}
seckill:token:{token}
auth:token:blacklist:{jti}
product:detail:{productId}
```

## 八、日志与追踪字段

每条业务日志须包含（见可观测性文档）：

`timestamp`、`level`、`service`、`traceId`、`requestId`、`userId`（如有）、`uri`、`durationMs`、`code`

`traceId` 由 OpenTelemetry 注入，与 `X-Request-Id` 并存。
